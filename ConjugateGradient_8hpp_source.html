14 /*!
15  * \ingroup LinearSolverGroup
16  * \brief A conjugate gradient solver for linear systems of equations \f$Ax=b\f$
17  * where the operator \f$A\f$ is symmetric.
18  *
19  * \details The only operation we need to supply to the algorithm is the
20  * result of the operation \f$A(p)\f$ (see \ref LinearSolverGroup) that in the
21  * case of the conjugate gradient algorithm must be symmetric. Each invocation
22  * of the `perform_step` action expects that \f$A(p)\f$ has been computed in a
23  * preceding action and stored in the DataBox as
24  * %db::add_tag_prefix<LinearSolver::Tags::OperatorAppliedTo,
25  * db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>>.
26  *
27  * Note that the operand \f$p\f$ for which \f$A(p)\f$ needs to be computed is
28  * not the field \f$x\f$ we are solving for but
29  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`. This field is
30  * initially set to the residual \f$r = b - A(x_0)\f$ where \f$x_0\f$ is the
31  * initial value of the `FieldsTag`.
32  *
33  * When the `perform_step` action is invoked after the operator action
34  * \f$A(p)\f$ has been computed and stored in the DataBox, the conjugate
35  * gradient algorithm implemented here will converge the field \f$x\f$ towards
36  * the solution and update the operand \f$p\f$ in the process. This requires
37  * two reductions over all elements that are received by a `ResidualMonitor`
38  * singleton parallel component, processed, and then broadcast back to all
39  * elements. The actions are implemented in the `cg_detail` namespace and
40  * constitute the full algorithm in the following order:
41  * 1. `PerformStep` (on elements): Compute the inner product \f$\langle p,
42  * A(p)\rangle\f$ and reduce.
43  * 2. `ComputeAlpha` (on `ResidualMonitor`): Compute
44  * \f$\alpha=\frac{r^2}{\langle p, A(p)\rangle}\f$ and broadcast.
45  * 3. `UpdateFieldValues` (on elements): Update \f$x\f$ and \f$r\f$, then
46  * compute the inner product \f$\langle r, r\rangle\f$ and reduce to find the
47  * new \f$r^2\f$.
48  * 4. `UpdateResidual` (on `ResidualMonitor`): Store the new \f$r^2\f$ and
49  * broadcast the ratio of the new and old \f$r^2\f$, as well as a termination
50  * flag if the `LinearSolver::Tags::ConvergenceCriteria` are met.
51  * 5. `UpdateOperand` (on elements): Update \f$p\f$.
52  *
53  * \see Gmres for a linear solver that can invert nonsymmetric operators
54  * \f$A\f$.
55  */
56 template <typename Metavariables, typename FieldsTag>
57 struct ConjugateGradient {
58  /*!
59  * \brief The parallel components used by the conjugate gradient linear solver
60  */
61  using component_list =
62  tmpl::list<cg_detail::ResidualMonitor<Metavariables, FieldsTag>>;
63 
64  /*!
65  * \brief Initialize the tags used by the conjugate gradient linear solver.
66  *
67  * Since we have not started iterating yet, we initialize the state _before_
68  * the first iteration. So `LinearSolver::Tags::IterationId` is undefined at
69  * this point and `Tags::Next<LinearSolver::Tags::IterationId>` is the initial
70  * step number. Invoke `prepare_step` to advance the state to the first
71  * iteration.
72  *
73  * \warning This action involves a blocking reduction, so it is a global
74  * synchronization point.
75  *
76  * With:
77  * - `operand_tag` =
78  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`
79  * - `operator_tag` =
80  * `db::add_tag_prefix<LinearSolver::Tags::OperatorAppliedTo, operand_tag>`
81  * - `residual_tag` =
82  * `db::add_tag_prefix<LinearSolver::Tags::Residual, FieldsTag>`
83  *
84  * DataBox changes:
85  * - Adds:
86  * * `LinearSolver::Tags::IterationId`
87  * * `Tags::Next<LinearSolver::Tags::IterationId>`
88  * * `residual_tag`
89  * * `LinearSolver::Tags::HasConverged`
90  * - Removes: nothing
91  * - Modifies:
92  * * `operand_tag`
93  *
94  * \note The `operand_tag` must already be present in the DataBox and is set
95  * to its initial value here. It is typically added to the DataBox by the
96  * system, which uses it to compute the `operator_tag` in each step. Also the
97  * `operator_tag` is typically added to the DataBox by the system, but does
98  * not need to be initialized until it is computed for the first time in the
99  * first step of the algorithm.
100  */
101  using initialize_element = cg_detail::InitializeElement<FieldsTag>;
102 
103  /*!
104  * \brief Reset the linear solver to its initial state.
105  *
106  * With:
107  * - `operand_tag` =
108  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`
109  * - `residual_tag` =
110  * `db::add_tag_prefix<LinearSolver::Tags::Residual, FieldsTag>`
111  *
112  * DataBox changes:
113  * - Adds: nothing
114  * - Removes: nothing
115  * - Modifies:
116  * * `LinearSolver::Tags::IterationId`
117  * * `residual_tag`
118  * * `LinearSolver::Tags::HasConverged`
119  * * `operand_tag`
120  *
121  * \see `initialize_element`
122  */
123  using reinitialize_element =
124  cg_detail::InitializeElement<FieldsTag,
125  ::Initialization::MergePolicy::Overwrite>;
126 
127  // Compile-time interface for observers
128  using observed_reduction_data_tags = observers::make_reduction_data_tags<
129  tmpl::list<observe_detail::reduction_data>>;
130 
131  /*!
132  * \brief Advance the linear solver to the next iteration.
133  *
134  * DataBox changes:
135  * - Adds: nothing
136  * - Removes: nothing
137  * - Modifies:
138  * * `LinearSolver::Tags::IterationId`
139  * * `Tags::Next<LinearSolver::Tags::IterationId>`
140  */
141  using prepare_step = cg_detail::PrepareStep;
142 
143  /*!
144  * \brief Perform an iteration of the conjugate gradient linear solver.
145  *
146  * \warning This action involves a blocking reduction, so it is a global
147  * synchronization point.
148  *
149  * With:
150  * - `operand_tag` =
151  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`
152  * - `residual_tag` =
153  * `db::add_tag_prefix<LinearSolver::Tags::Residual, FieldsTag>`
154  *
155  * DataBox changes:
156  * - Adds: nothing
157  * - Removes: nothing
158  * - Modifies:
159  * * `FieldsTag`
160  * * `operand_tag`
161  * * `residual_tag`
162  * * `LinearSolver::Tags::HasConverged`
163  */
164  using perform_step = cg_detail::PerformStep<FieldsTag>;
165 };

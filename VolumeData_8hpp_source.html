22 /*!
23  * \ingroup HDF5Group
24  * \brief A volume data subfile written inside an H5 file.
25  *
26  * The volume data inside the subfile can be of any dimensionality greater than
27  * zero. This means that in a 3D simulation, data on 2-dimensional surfaces are
28  * written as a VolumeData subfile. Data can be written using the
29  * `write_volume_data()` method. An integral observation id is used to keep
30  * track of the observation instance at which the data is written, and
31  * associated with it is a floating point observation value, such as the
32  * simulation time at which the data was written. The observation id will
33  * generally be the result of hashing the temporal identifier used for the
34  * simulation.
35  *
36  * The data stored in the subfile are the tensor components passed to the
37  * `write_volume_data()` method as a `std::vector<ExtentsAndTensorVolumeData>`.
38  * The name of each tensor component must follow the format
39  * `GRID_NAME/TENSOR_NAME_COMPONENT`, e.g. `Element0/T_xx`. Typically the
40  * `GRID_NAME` should be the output of the stream operator of the spatial ID of
41  * the parallel component element sending the data to be observed. For example,
42  * in the case of a dG evolution where the spatial IDs are `ElementId`s, the
43  * grid names would be of the form `[B0,(L2I3,L2I3,L2I3)]`. The data are
44  * written contiguously inside of the H5 subfile, in that each tensor
45  * component has a single dataset which holds all of the data from
46  * all elements, e.g. a tensor component `T_xx` which is found on all grids
47  * appears in the path `H5_FILE_NAME/element_data.vol/T_xx` and that is where
48  * all of the `T_xx` data from all of the grids resides. Note that coordinates
49  * must be written as tensors in order to visualize the data in ParaView,
50  * Visit, etc. In order to reconstruct which data came from which grid,
51  * the `get_grid_names()`, and `get_extents()` methods list the grids
52  * and their extents in the order which they and the data were written.
53  * For example, if the first grid has name `GRID_NAME` with extents
54  * `{2, 2, 2}`, it was responsible for contributing the first 2*2*2 = 8 grid
55  * points worth of data in each tensor dataset.
56  *
57  * \warning Currently the topology of the grids is assumed to be tensor products
58  * of lines, i.e. lines, quadrilaterals, and hexahedrons. However, this can be
59  * extended in the future. If support for more topologies is required, please
60  * file an issue.
61  */

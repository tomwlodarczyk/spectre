      69             :  * 1. A Parallel::Algorithms::Singleton where there is only one
      70             :  * in the entire execution of the program.
      71             :  * 2. A Parallel::Algorithms::Array which holds zero or more
      72             :  * elements each of which is a distributed object on some core. An array can
      73             :  * grow and shrink in size dynamically if need be and can also be bound to
      74             :  * another array. That is, the bound array has the same number of elements as
      75             :  * the array it is bound to, and elements with the same ID are on the same core.
      76             :  * 3. A Parallel::Algorithms::Group, which is an array but there is
      77             :  * one element per core and they are not able to be moved around between cores.
      78             :  * These are typically useful for gathering data from array elements on their
      79             :  * core, and then processing or reducing it.
      80             :  * 4. A Parallel::Algorithms::Nodegroup, which is similar to a
      81             :  * group except that there is one element per node. For Charm++ SMP (shared
      82             :  * memory parallelism) builds a node corresponds to the usual definition of a
      83             :  * node on a supercomputer. However, for non-SMP builds nodes and cores are
      84             :  * equivalent. An important difference between groups and nodegroups is that
      85             :  * entry methods (remote calls to functions) are not threadsafe on nodegroups.
      86             :  * It is up to the person writing the Actions that will be executed on the
      87             :  * Nodegroup Algorithm to ensure they are threadsafe.
      88             :  *
      89             :  * ### What is an Algorithm?

      21             : //============================================================================
      22             : // Note that SPHEREPACK (which is wrapped by YlmSpherepack) takes
      23             : // n_theta and n_phi as input, and is ok with arbitrary values of
      24             : // n_theta and n_phi.  SPHEREPACK computes the maximum Ylm l and m
      25             : // using the formulas l_max=n_theta-1 and m_max=min(l_max,n_phi/2).
      26             : //
      27             : // However, some combinations of n_theta and n_phi have strange properties:
      28             : // - The maximum m that is AT LEAST PARTIALLY represented by (n_theta,n_phi)
      29             : //   is std::min(n_theta-1,n_phi/2). This is called m_max here. But an arbitrary
      30             : //   (n_theta,n_phi) does not necessarily fully represent all m's up to m_max,
      31             : //   because sin(m_max phi) might be zero at all collocation points, and
      32             : //   therefore sin(m_max phi) might not be representable on the grid.
      33             : // - The largest m that is fully represented by (n_theta,n_phi) is
      34             : //   m_max_represented = std::min(n_theta-1,(n_phi-1)/2).
      35             : // - Therefore, if n_phi is odd,  m_max = m_max_represented,
      36             : //              if n_phi is even, m_max = m_max_represented+1.
      37             : // - To remedy this situation, we choose YlmSpherepack to take as arguments
      38             : //   l_max and m_max, instead of n_theta and n_phi.
      39             : //   We then choose
      40             : //      n_theta = l_max+1
      41             : //      n_phi   = 2*m_max+1
      42             : //   This ensures that m_max = m_max_represented
      43             : //   (as opposed to m_max = m_max_represented+1)
      44             : //============================================================================

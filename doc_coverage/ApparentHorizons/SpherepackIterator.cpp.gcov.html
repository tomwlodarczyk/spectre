       5             : // SPHEREPACK memory layout
       6             : //
       7             : // For a surface of topology S2 with a spherical harmonic basis and a
       8             : // grid of n_th x n_ph points, the largest (partially) representable
       9             : // spherical-harmonic Ylm modes (l,m) are:
      10             : // * l_max = n_th-1
      11             : // * m_max = min(l_max, n_ph/2) [integer division]
      12             : //
      13             : // Given collocation values on this n_th x n_ph grid, SPHEREPACK
      14             : // stores the spectral coefficients in an array of length
      15             : // (l_max+1)*(m_max+1).  The layout of this array is, in a
      16             : // 2-dimensional representation
      17             : //
      18             : //                            m_max+1
      19             : //     -----------------------------------------------------
      20             : //     | (0,0)         x       x            x
      21             : //     | (1,0)       (1,1)     x            x         ARRAY
      22             : //     | (2,0)       (2,1)   (2,2)          x          "a"
      23             : //     |   :           :       :            x
      24             : //     | (m_max,  0)  ...            (m_max,   m_max)
      25             : //     | (m_max+1,0)  ...            (m_max+1, m_max)
      26             : //     |    :                  :            :
      27             : //     | (l_max, 0)                  (l_max,   m_max)
      28             : //     -----------------------------------------------------
      29             : //     |    x        x          x          x
      30             : //     |    x      (1,1)        x          x          ARRAY
      31             : //     |    x      (2,1)      (2,2)        x           "b"
      32             : //     |    :        :          :          x
      33             : //     |    x    (m_max,  1)..      (m_max,   m_max)
      34             : //     |    x    (m_max+1,1)..      (m_max+1, m_max)
      35             : //     |    :         :               :
      36             : //     |    x    (l_max,  1)..      (l_max,   m_max)
      37             : //     -----------------------------------------------------
      38             : //
      39             : //
      40             : // In this table,
      41             : //
      42             : // * There are 2*n_th rows and m_max+1 columns. The SPHEREPACK array
      43             : //   stores this data "row by row", i.e. the numbers across a row vary
      44             : //   fastest.
      45             : //
      46             : // * There are no entries with negative m. SPHEREPACK uses positive m
      47             : //   only, but with complex spectral coefficients. The portions "a"
      48             : //   and "b" of the SPHEREPACK array hold the real and imaginary parts
      49             : //   of the spectral coefficients.
      50             : //
      51             : // * Each "x" stands for an entry that is not referenced by SPHEREPACK
      52             : //   (so SPHEREPACK wastes some storage space).  There is also one
      53             : //   complete unreferenced row in the middle: the first row of the "b"
      54             : //   array.
      55             : //
      56             : // Note that SPHEREPACK accepts arbitrary values of n_th and n_ph as
      57             : // input, and then computes the corresponding l_max, m_max.  However,
      58             : // some combinations of n_th and n_ph have strange properties:
      59             : // * The maximum M that is AT LEAST PARTIALLY represented by (n_th,n_ph)
      60             : //   is std::min(n_th-1,n_ph/2). This is called m_max here. But an arbitrary
      61             : //   (n_th,n_ph) does not necessarily fully represent all m's up to m_max,
      62             : //   because sin(m_max phi) might be zero at all collocation points, and
      63             : //   therefore sin(m_max phi) might not be representable on the grid.
      64             : // * The largest m that is fully represented by (n_th,n_ph) is
      65             : //   m_max_represented = std::min(n_th-1,(n_ph-1)/2).
      66             : // * Therefore, if n_ph is odd,  m_max = m_max_represented,
      67             : //              if n_ph is even, m_max = m_max_represented+1.
      68             : // * To fully represent a desired (l_max, m_max), the grid resolution
      69             : //   should satisfy
      70             : //     n_th = l_max+1
      71             : //     n_ph = 2*m_max_represented+1
      72             : //   which ensures that m_max = m_max_represented.

      64             : # What is SpECTRE? {#intro_sec}
      65             : 
      66             : SpECTRE is an open-source code for multi-scale, multi-physics problems
      67             : in astrophysics and gravitational physics. In the future, we hope that
      68             : it can be applied to problems across discipline boundaries in fluid
      69             : dynamics, geoscience, plasma physics, nuclear physics, and
      70             : engineering. It runs at petascale and is designed for future exascale
      71             : computers.
      72             : 
      73             : SpECTRE is being developed in support of our collaborative Simulating
      74             : eXtreme Spacetimes (SXS) research program into the multi-messenger
      75             : astrophysics of neutron star mergers, core-collapse supernovae, and
      76             : gamma-ray bursts.
      77             : 
      78             : ## Navigating the Documentation {#navigate_documentation_sec}
      79             : 
      80             : The SpECTRE documentation is organized into tutorials, developer guides, groups
      81             : of related code, namespaces, and files for easier navigation. These can all be
      82             : accessed by links in the menu bar at the top.
      83             : 
      84             : - For instructions on **installing SpECTRE** on personal computers and clusters
      85             :   consult the \ref installation "Installation" and \ref installation_on_clusters
      86             :   "Installation on clusters" pages, respectively.
      87             : - If you are looking to **run simulations with SpECTRE** we recommend starting
      88             :   with the \ref tutorials "User Tutorials". The tutorials are designed to get
      89             :   users up and running with a simulation, as well as analyzing and visualizing
      90             :   the output.
      91             : - For people looking to **contribute to SpECTRE** there are tutorials on the
      92             :   \ref dev_guide "Dev Guide" page. For instance, the dev guide details the \ref
      93             :   code_review_guide "code review guidelines", how to \ref writing_unit_tests
      94             :   "write unit tests", how to \ref writing_good_dox "write documentation", and
      95             :   also provides information about C++ and the philosophy behind SpECTRE
      96             :   development.
      97             : - The [Topics](modules.html) section under the _Reference_ menu item contains
      98             :   groups of related code (managed through doxygen groups). For example, there is
      99             :   a group for all the data structures we use, a group for utility functions and
     100             :   classes, as well as groups for coordinate maps, domain creation, and many
     101             :   others. The [Topics](modules.html) are designed to help developers discover
     102             :   existing functionality so that things are not re-implemented several times.
     103             : - You can also get an overview of the code base by namespace by visiting the
     104             :   [Namespaces](namespaces.html) section under the _Reference_ menu item.
     105             : - Finally, it is also possible to browse the repository by files under the
     106             :   [Files](files.html) menu item, though it is recommended that in that case you
     107             :   browse the [GitHub repository](https://github.com/sxs-collaboration/spectre)
     108             :   directly.

       5             : # Using SpECTRE's Python modules {#spectre_using_python}
       6             : 
       7             : Some classes and functions from SpECTRE have Python bindings to make it easier
       8             : to visualize data, write test code, and provide an introduction to numerical
       9             : relativity without needing to delve into C++.
      10             : 
      11             : ## Installing the SpECTRE Python modules
      12             : 
      13             : First, build SpECTRE with Python bindings enabled by appending the
      14             : `-D BUILD_PYTHON_BINDINGS=ON` flag to the `cmake` command. You will find that
      15             : a `BUILD_DIR/bin/python` directory is created that contains the Python modules.
      16             : Then, you can install the modules into your Python environment in
      17             : _development mode_, which means they are symlinked so that changes to the
      18             : modules will be reflected in your Python environment, with the command
      19             : `pip install -e path/to/bin/python`. Alternatively, remove the `-e` flag to
      20             : install the modules normally. You can do this in any Python environment that
      21             : supports `pip`, for instance in a
      22             : [`virtualenv`/`venv`](https://docs.python.org/3/tutorial/venv.html) or in an
      23             : [Anaconda](https://www.anaconda.com/distribution/) environment. You can also get
      24             : access to the SpECTRE Python modules by adding  `BUILD_DIR/bin/python` to your
      25             : `PYTHONPATH`. This is done automatically by sourcing the `LoadPython.sh` file
      26             : with the command `. BUILD_DIR/bin/LoadPython.sh`.

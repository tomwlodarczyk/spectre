       7             : <para><heading level="2">CMake and Directory Layout</heading>
       8             : </para><para>To allow users to analyze output from simulations and take advantage of SpECTRE&apos;s data structures and functions in python, bindings must sometimes be written. SpECTRE uses <ulink url="https://www.boost.org/doc/libs/1_68_0/libs/python/doc/html/index.html">Boost.Python</ulink> to aid with generating the bindings. The C++ code for the bindings should generally go in a <computeroutput>Python</computeroutput> subdirectory. For example, the bindings for the DataStructures library would go in <computeroutput>src/DataStructures/Python/</computeroutput>. SpECTRE provides the <computeroutput>spectre_python_add_module</computeroutput> CMake function to make adding a new python module, be it with or without bindings, easy. The python bindings are built only if <computeroutput>-D BUILD_PYTHON_BINDINGS=ON</computeroutput> is passed when invoking cmake.</para><para>The function <computeroutput>spectre_python_add_module</computeroutput> takes as its first argument the module, in our case <computeroutput>DataStructures</computeroutput>. The bindings library will be prefixed with <computeroutput>Py</computeroutput>, and therefore be called <computeroutput>PyDataStructures</computeroutput>. Optionally, a list of <computeroutput>SOURCES</computeroutput> can be passed to the CMake function. If the python module will only consist of python files, then the <computeroutput>SOURCES</computeroutput> option should not be specified. Python files that should be part of the module can be passed with the keyword <computeroutput>PYTHON_FILES</computeroutput>, e.g. <computeroutput>PYTHON_FILES Hello.py HelloWorld.py</computeroutput>. Finally, the <computeroutput>MODULE_PATH</computeroutput> named argument can be passed with a string that is the path to where the module should be. For example, <computeroutput>MODULE_PATH &quot;submodule0/submodule1/&quot;</computeroutput> would mean the module is accessed from python using <computeroutput>import spectre.submodule0.submodule1.MODULE_NAME</computeroutput>.</para><para>Here is a complete example of how to call the <computeroutput>spectre_python_add_module</computeroutput> function:</para><para><programlisting><codeline><highlight class="normal">spectre_python_add_module(</highlight></codeline>

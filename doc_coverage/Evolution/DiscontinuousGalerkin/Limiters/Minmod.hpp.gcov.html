      79             : ///
      80             : /// The MUSCL and \f$\Lambda\Pi^1\f$ limiters are both intended for use on
      81             : /// piecewise-linear solutions, i.e., on linear-order elements with two points
      82             : /// per dimension. These limiters operate by reducing the spatial slope of the
      83             : /// tensor components if the data look like they may contain oscillations.
      84             : /// Between these two, MUSCL is more dissipative --- it more aggressively
      85             : /// reduces the slopes of the data, so it may better handle strong shocks, but
      86             : /// it correspondingly produces the most broadening of features in the solution.
      87             : ///
      88             : /// Note that we do not _require_ the MUSCL and \f$\Lambda\Pi^1\f$ limiters to
      89             : /// be used on linear-order elements. However, when they are used on a
      90             : /// higher-resolution grid, the limiters act to linearize the solution (by
      91             : /// discarding higher-order mode content) whether or not the slopes must be
      92             : /// reduced.
      93             : ///
      94             : /// The \f$\Lambda\Pi^N\f$ limiter is intended for use with higher-order

      97             : /// \f$\Lambda\Pi^1\f$: the post-limiter solution is the linearized solution of

      99             : /// original (higher-order) data in the case that the slopes are acceptable.

     102             : /// means" (TVBM) correction is implemented, enabling the limiter to avoid

     105             : /// linearize) on elements where the slope is less than \f$m h^2\f$, where
     106             : /// \f$m\f$ is the TVBM constant and \f$h\f$ is the size of the DG element.

     108             : /// The limiter acts in the `Frame::Logical` coordinates, because in these

     116             : ///    typically small.

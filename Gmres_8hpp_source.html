15 /*!
16  * \ingroup LinearSolverGroup
17  * \brief A GMRES solver for nonsymmetric linear systems of equations
18  * \f$Ax=b\f$.
19  *
20  * \details The only operation we need to supply to the algorithm is the
21  * result of the operation \f$A(p)\f$ (see \ref LinearSolverGroup). Each
22  * invocation of the `perform_step` action expects that \f$A(q)\f$ has been
23  * computed in a preceding action and stored in the DataBox as
24  * %db::add_tag_prefix<LinearSolver::Tags::OperatorAppliedTo,
25  * db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>>.
26  *
27  * Note that the operand \f$q\f$ for which \f$A(q)\f$ needs to be computed is
28  * not the field \f$x\f$ we are solving for but
29  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`. This field is
30  * initially set to the residual \f$q_0 = b - A(x_0)\f$ where \f$x_0\f$ is the
31  * initial value of the `FieldsTag`.
32  *
33  * When the `perform_step` action is invoked after the operator action
34  * \f$A(q)\f$ has been computed and stored in the DataBox, the GMRES algorithm
35  * implemented here will converge the field \f$x\f$ towards the solution and
36  * update the operand \f$q\f$ in the process. This requires reductions over
37  * all elements that are received by a `ResidualMonitor` singleton parallel
38  * component, processed, and then broadcast back to all elements. Since the
39  * reductions are performed to find a vector that is orthogonal to those used in
40  * previous steps, the number of reductions increases linearly with iterations.
41  * No restarting mechanism is currently implemented. The actions are implemented
42  * in the `gmres_detail` namespace and constitute the full algorithm in the
43  * following order:
44  * 1. `PerformStep` (on elements): Start an Arnoldi orthogonalization by
45  * computing the inner product between \f$A(q)\f$ and the first of the
46  * previously determined set of orthogonal vectors.
47  * 2. `StoreOrthogonalization` (on `ResidualMonitor`): Keep track of the
48  * computed inner product in a Hessenberg matrix, then broadcast.
49  * 3. `OrthogonalizeOperand` (on elements): Proceed with the Arnoldi
50  * orthogonalization by computing inner products and reducing to
51  * `StoreOrthogonalization` on the `ResidualMonitor` until the new orthogonal
52  * vector is constructed. Then compute its magnitude and reduce.
53  * 4. `StoreFinalOrthogonalization` (on `ResidualMonitor`): Perform a QR
54  * decomposition of the Hessenberg matrix to produce a residual vector.
55  * Broadcast to `NormalizeOperandAndUpdateField` along with a termination
56  * flag if the `LinearSolver::Tags::ConvergenceCriteria` are met.
57  * 5. `NormalizeOperandAndUpdateField` (on elements): Set the operand \f$q\f$ as
58  * the new orthogonal vector and normalize. Use the residual vector and the set
59  * of orthogonal vectors to determine the solution \f$x\f$.
60  *
61  * \see ConjugateGradient for a linear solver that is more efficient when the
62  * linear operator \f$A\f$ is symmetric.
63  */
64 template <typename Metavariables, typename FieldsTag>
65 struct Gmres {

67  * \brief The parallel components used by the GMRES linear solver

72  /*!
73  * \brief Initialize the tags used by the GMRES linear solver.
74  *
75  * Since we have not started iterating yet, we initialize the state _before_
76  * the first iteration. So `LinearSolver::Tags::IterationId` is undefined at
77  * this point and `Tags::Next<LinearSolver::Tags::IterationId>` is the initial
78  * step number. Invoke `prepare_step` to advance the state to the first
79  * iteration.
80  *
81  * \warning This action involves a blocking reduction, so it is a global
82  * synchronization point.
83  *
84  * With:
85  * - `initial_fields_tag` =
86  * `db::add_tag_prefix<LinearSolver::Tags::Initial, FieldsTag>`
87  * - `operand_tag` =
88  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`
89  * - `operator_tag` =
90  * `db::add_tag_prefix<LinearSolver::Tags::OperatorAppliedTo, operand_tag>`
91  * - `orthogonalization_iteration_id_tag` =
92  * `db::add_tag_prefix<LinearSolver::Tags::Orthogonalization,
93  * LinearSolver::Tags::IterationId>`
94  * - `basis_history_tag` =
95  * `LinearSolver::Tags::KrylovSubspaceBasis<FieldsTag>`
96  *
97  * DataBox changes:
98  * - Adds:
99  * * `LinearSolver::Tags::IterationId`
100  * * `Tags::Next<LinearSolver::Tags::IterationId>`
101  * * `initial_fields_tag`
102  * * `orthogonalization_iteration_id_tag`
103  * * `basis_history_tag`
104  * * `LinearSolver::Tags::HasConverged`
105  * - Removes: nothing
106  * - Modifies:
107  * * `operand_tag`
108  *
109  * \note The `operand_tag` must already be present in the DataBox and is set
110  * to its initial value here. It is typically added to the DataBox by the
111  * system, which uses it to compute the `operator_tag` in each step. Also the
112  * `operator_tag` is typically added to the DataBox by the system, but does
113  * not need to be initialized until it is computed for the first time in the
114  * first step of the algorithm.
115  */
116  using initialize_element = gmres_detail::InitializeElement<FieldsTag>;
117 
118  /*!
119  * \brief Reset the linear solver to its initial state.
120  *
121  * Uses:
122  * - System:
123  * * `fields_tag`
124  *
125  * With:
126  * - `initial_fields_tag` =
127  * `db::add_tag_prefix<LinearSolver::Tags::Initial, fields_tag>`
128  * - `operand_tag` =
129  * `db::add_tag_prefix<LinearSolver::Tags::Operand, fields_tag>`
130  * - `orthogonalization_iteration_id_tag` =
131  * `db::add_tag_prefix<LinearSolver::Tags::Orthogonalization,
132  * LinearSolver::Tags::IterationId>`
133  * - `basis_history_tag` =
134  * `LinearSolver::Tags::KrylovSubspaceBasis<fields_tag>`
135  *
136  * DataBox changes:
137  * - Adds: nothing
138  * - Removes: nothing
139  * - Modifies:
140  * * `LinearSolver::Tags::IterationId`
141  * * `initial_fields_tag`
142  * * `orthogonalization_iteration_id_tag`
143  * * `basis_history_tag`
144  * * `LinearSolver::Tags::HasConverged`
145  * * `operand_tag`
146  *
147  * \see `initialize_element`
148  */
149  using reinitialize_element =
150  gmres_detail::InitializeElement<FieldsTag,
151  ::Initialization::MergePolicy::Overwrite>;
152 
153  // Compile-time interface for observers
154  using observed_reduction_data_tags = observers::make_reduction_data_tags<
155  tmpl::list<observe_detail::reduction_data>>;
156 
157  /*!
158  * \brief Advance the linear solver to the next iteration.
159  *
160  * DataBox changes:
161  * - Adds: nothing
162  * - Removes: nothing
163  * - Modifies:
164  * * `LinearSolver::Tags::IterationId`
165  * * `Tags::Next<LinearSolver::Tags::IterationId>`
166  * * `orthogonalization_iteration_id_tag`
167  */
168  using prepare_step = gmres_detail::PrepareStep;
169 
170  /*!
171  * \brief Perform an iteration of the GMRES linear solver.
172  *
173  * \warning This action involves a blocking reduction, so it is a global
174  * synchronization point.
175  *
176  * With:
177  * - `operand_tag` =
178  * `db::add_tag_prefix<LinearSolver::Tags::Operand, FieldsTag>`
179  * - `orthogonalization_iteration_id_tag` =
180  * `db::add_tag_prefix<LinearSolver::Tags::Orthogonalization,
181  * LinearSolver::Tags::IterationId>`
182  * - `basis_history_tag` =
183  * `LinearSolver::Tags::KrylovSubspaceBasis<FieldsTag>`
184  *
185  * DataBox changes:
186  * - Adds: nothing
187  * - Removes: nothing
188  * - Modifies:
189  * * `FieldsTag`
190  * * `operand_tag`
191  * * `orthogonalization_iteration_id_tag`
192  * * `basis_history_tag`
193  * * `LinearSolver::Tags::HasConverged`
194  */
195  using perform_step = gmres_detail::PerformStep<FieldsTag>;
196 };

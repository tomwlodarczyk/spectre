66 /*!
67  * \ingroup DataBoxGroup
68  * \brief Tags that are base tags, i.e. a simple or compute tag must derive
69  * off them for them to be useful
70  *
71  * Base tags do not need to contain type information, unlike simple
72  * tags which must contain the type information. Base tags are designed so
73  * that retrieving items from the DataBox or setting argument tags in compute
74  * items can be done without any knowledge of the type of the item.
75  *
76  * To use the base mechanism the base tag must inherit off of
77  * `BaseTag` and NOT `SimpleTag`. This is very important for the
78  * implementation. Inheriting off both and not making the tag either a simple
79  * item or compute item is undefined behavior and is likely to end in extremely
80  * complicated compiler errors.

113 /*!
114  * \ingroup DataBoxGroup
115  * \brief Marks a DataBoxTag as being a compute item that executes a function
116  *
117  * \details
118  * Compute items come in two forms: mutating and non-mutating. Mutating
119  * compute items modify a stored value in order to reduce the number of memory
120  * allocations done. For example, if a function would return a `Variables` or
121  * `Tensor<DataVector...>` and is called every time step, then it would be
122  * preferable to use a mutating compute item so that the values in the already
123  * allocated memory can just be changed.
124  * In contrast, non-mutating compute items simply return the new value after a
125  * call (if the value is out-of-date), which is fine for infrequently called
126  * compute items or ones that do not allocate data on the heap.
127  *
128  * A compute item tag contains a member named `function` that is either a
129  * function pointer, or a static constexpr function. The compute item tag
130  * must also have a `label`, same as the DataBox tags, and a type alias
131  * `argument_tags` that is a typelist of the tags that will
132  * be retrieved from the DataBox and whose data will be passed to the function
133  * (pointer). Mutating compute item tags must also contain a type alias named
134  * `return_type` that is the type the function is mutating. The type must be
135  * default constructible.
136  *
137  * \example
138  * Most non-mutating compute item tags will look similar to:
139  * \snippet Test_DataBox.cpp databox_compute_item_tag_example
140  * Note that the arguments can be empty:
141  * \snippet Test_DataBox.cpp compute_item_tag_no_tags
142  *
143  * Mutating compute item tags are of the form:
144  * \snippet Test_DataBox.cpp databox_mutating_compute_item_tag
145  * where the function is:
146  * \snippet Test_DataBox.cpp databox_mutating_compute_item_function
147  *
148  * You can also have `function` be a function instead of a function pointer,
149  * which offers a lot of simplicity for very simple compute items.
150  * \snippet Test_DataBox.cpp compute_item_tag_function
151  *
152  * \see DataBox SimpleTag tag_name PrefixTag
153  */
154 struct ComputeTag {};
